# Demo GIF + install.ps1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship a ~30 s `assets/demo.gif` at the top of both READMEs (rendered from a checked-in vhs tape via GitHub Actions) and an `install.ps1` giving Windows install parity with `install.sh`.

**Architecture:** Two independent deliverables, no CLI code changes. The GIF is a staged replay: `demo/demo.tape` (vhs) types the user's lines into `demo/play.sh`, a fake pre-scripted agent session whose substantive content is taken verbatim from `examples/micrograd` (evidence-bound, r1–r6). A `workflow_dispatch` GitHub Action renders the tape and commits `assets/demo.gif`. `install.ps1` mirrors `install.sh` step-for-step.

**Tech Stack:** PowerShell 5.1, POSIX sh, charmbracelet/vhs (`charmbracelet/vhs-action@v2`), GitHub Actions.

**Spec:** `docs/superpowers/specs/2026-07-02-demo-gif-and-install-ps1-design.md`

## Global Constraints

- Never edit the user's environment (PATH, shell rc, registry) — print the exact line instead.
- Demo content honesty: every substantive answer line in the demo traces to `examples/micrograd` (`mask.md`, `knowledge/engine.md`, `sources.json` ids r1–r6). No invented facts or citations.
- `install.ps1` must run on Windows PowerShell 5.1: no `&&`/`||`, no ternary. **ASCII-only file content** (PS 5.1 misreads BOM-less UTF-8; the sh version's `✅`/`⚠`/`…` become `OK.`/`NOTE:`/`...`).
- `demo/play.sh` is demo-only: plain POSIX sh, imports nothing from `src/`.
- Installer scripts have no unit tests (parity with `install.sh`); verification is by execution. `bun run typecheck` / `bun test` still must pass before each commit (nothing here should affect them).
- Repo convention: one minimal, verifiable commit per change.

- [ ] **Step 1: Write `install.ps1`**

```powershell
#Requires -Version 5.1
# mask installer (Windows) -- run from a clone:  .\install.ps1
# Installs deps and drops a `mask.cmd` launcher on your PATH that runs the CLI
# from this checkout (so `git pull` updates it; no rebuild, no MASK_FRAMEWORK needed).
#
# Override the install dir:  .\install.ps1 -InstallDir C:\some\bin   (or $env:INSTALL_DIR)
param(
  [string]$InstallDir
)

$ErrorActionPreference = 'Stop'

$Repo = $PSScriptRoot
if (-not $InstallDir) {
  if ($env:INSTALL_DIR) { $InstallDir = $env:INSTALL_DIR }
  else { $InstallDir = Join-Path $env:USERPROFILE '.local\bin' }
}
$Bin = Join-Path $InstallDir 'mask.cmd'

if (-not (Get-Command bun -ErrorAction SilentlyContinue)) {
  [Console]::Error.WriteLine('mask: Bun is required but not found. Install it from https://bun.sh then re-run .\install.ps1')
  exit 1
}

Write-Host 'mask: installing dependencies...'
Push-Location $Repo
try {
  bun install --frozen-lockfile | Out-Null
  $installExit = $LASTEXITCODE
} finally {
  Pop-Location
}
if ($installExit -ne 0) { exit $installExit }

New-Item -ItemType Directory -Force $InstallDir | Out-Null
$launcher = @"
@echo off
rem mask launcher -- generated by install.ps1; runs the CLI from the checkout.
bun run "$Repo\src\cli.ts" %*
"@
Set-Content -Path $Bin -Value $launcher -Encoding Ascii
Write-Host "mask: installed launcher -> $Bin"

# PATH check -- never edit the user's PATH; just print the exact line.
$dirs = $env:Path -split ';' | ForEach-Object { $_.TrimEnd('\') }
if ($dirs -contains $InstallDir.TrimEnd('\')) {
  Write-Host ''
  Write-Host 'OK. Try:  mask --version'
} else {
  Write-Host ''
  Write-Host "NOTE: $InstallDir is not on your PATH. Add it for your user account by"
  Write-Host 'running this once in PowerShell, then open a new terminal:'
  Write-Host ''
  Write-Host "    [Environment]::SetEnvironmentVariable('Path', [Environment]::GetEnvironmentVariable('Path','User') + ';$InstallDir', 'User')"
  Write-Host ''
  Write-Host 'Then:  mask --version'
}

Write-Host ''
Write-Host 'Next:  cd your-project; mask init --agent agents-md --out .'
Write-Host '       (Claude Code users: just  mask init)'
```

### Task 2: demo/play.sh — the staged session presenter

**Files:**
- Create: `demo/play.sh`

**Interfaces:**
- Consumes: content facts from `examples/micrograd/knowledge/engine.md` and `examples/micrograd/sources.json` (6 samples, ids r1–r6).
- Produces: an interactive script that prints a `❯ ` prompt, `read`s one line, and prints the scripted agent response — three rounds, then an end card. Task 3's tape types into these prompts.

- [ ] **Step 1: Write `demo/play.sh`**

```sh
#!/bin/sh
# demo presenter for the README GIF -- a pre-scripted "agent session".
# demo/demo.tape (vhs) types the user's lines into the prompts below; this
# script plays the agent's side. Substantive content (pipeline shape, sample
# ids, the answer, its [src:] citations) is sourced from examples/micrograd --
# see docs/superpowers/specs/2026-07-02-demo-gif-and-install-ps1-design.md.

p()   { printf '%s\n' "$1"; }
dim() { printf '\033[2m%s\033[0m\n' "$1"; }
ok()  { printf '\033[32m%s\033[0m\n' "$1"; }
prompt() { printf '\n\033[35m\xe2\x9d\xaf \033[0m'; }

# round 1: "distill karpathy/micrograd for me"
prompt; read -r _
sleep 0.6
dim '  ingest    github.com/karpathy/micrograd -> 6 samples'
sleep 0.9
dim '  reduce    -> digest (r1..r6)'
sleep 0.9
dim '  extract   conventions / architecture / APIs   [recipe: code]'
sleep 0.9
dim '  compile   -> subagent installed'
sleep 0.6
ok  '  distilled micrograd -- say "wear micrograd"'

# round 2: "wear micrograd"
prompt; read -r _
sleep 0.5
ok '  wearing micrograd  (code expert / karpathy/micrograd)'

# round 3: the question
prompt; read -r _
sleep 0.9
p ''
p '[micrograd] backward() builds a topological order with a post-order'
p 'DFS over _prev, seeds self.grad = 1, then runs each _backward() in'
p 'reversed(topo) [src:r3]. Gradients accumulate with += -- that is what'
p 'keeps a Value reused in several subexpressions correct [src:r3].'
sleep 3

# end card
printf '\n\n'
p '   Distill anything. Wear anyone.'
dim '   github.com/TTigger/mask'
sleep 3
```

(`\xe2\x9d\xaf` is `❯` as UTF-8 bytes so the file itself stays ASCII-safe; `printf` emits it correctly in the vhs terminal.)

- [ ] **Step 2: Verify it runs end-to-end locally**


Expected: exits 0 after ~14 s; output shows the four dim pipeline lines, the two green status lines, the 4-line answer containing `[src:r3]` twice, and the end card. No error output.

## As-built deviations

- (a) The workflow originally specified `charmbracelet/vhs-action@v2`, but that action has a broken ffmpeg installer (upstream issue vhs-action#459). The as-built workflow replaces it with direct pinned installs: `apt` for ffmpeg and ttyd, and charmbracelet/vhs 0.11.0 .deb verified by SHA256.

- (b) `demo/play.sh` uses a POSIX octal escape `\xe2\x9d\xaf` for the prompt `❯` symbol, because the plan's `\xHH` hex escape breaks under POSIX dash; also clears the screen at start with `clear`.

- (c) The actual rendered GIF is approximately 24 seconds, not the ~30 seconds estimated in the plan. The estimate double-counted script runtime that runs concurrently with tape sleep periods, reducing the net duration.
